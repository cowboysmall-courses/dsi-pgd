Predictive Model for Customer Churn: A Case Study in Banking¶

Objective¶

To develop a model/method for identifying customers who are likely to exit

Background:¶

Bank customer churn, also known as customer attrition, refers to the phenomenon where customers stop doing business with a bank or switch to another bank. Churn is a critical metric for banks as it directly impacts their customer base and revenue. The dataset represents bank customer information for churn analysis. Each row in the dataset corresponds to a specific customer and contains several features or attributes that describe them.

Splitting Data into Train and Test data sets¶

In [5]:
# Define the features and target variable
x_variables = ['CreditScore', 'Geography', 'Gender', 'Age', 'Tenure', 'Balance',
               'NumOfProducts', 'HasCrCard', 'IsActiveMember', 'EstimatedSalary']
X = data[x_variables]
y = data['Exited']

# Convert categorical variables into dummy variables
X_encoded = pd.get_dummies(X, columns=['Geography', 'Gender', 'HasCrCard', 'IsActiveMember'], drop_first=True)

# Split the data into training and testing sets
X_train, X_test, y_train, y_test = train_test_split(X_encoded, y, test_size=0.2, random_state=42)

Note :¶

When you are using LogisticRegression from scikit, it is recommended to standardize X variables to avoid the problem of non-convergence (which is not required while using logit())

Model 2. Naive Bayes¶

The scikit-learn library has two functions for Naive Bayes method:

  1. GaussianNB() for numeric predictors

  2. MultinomialNB() for categorical predictors

Here, we use GaussianNB() by converting categorical variables into numeric variables(dummy variables). This is a general industry practice. However one can convert numeric variable into categorical variable using MultinomialNB()
